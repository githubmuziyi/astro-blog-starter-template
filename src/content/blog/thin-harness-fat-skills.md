### 快速阅读
Garry Tan 通过对 Claude Code 源码的分析，揭示了真正让使用 AI 的工程师生产力暴涨 100 倍的秘密所在：并不是用了多牛的模型（因为大家用的模型是一样的），而是他们的底层架构——“薄框架，厚技能”（Thin Harness, Fat Skills）。核心思想是将智能处理“上移”到用 Markdown 编写的技能文件中，而将执行逻辑“下移”到完全确定性的工具中，中间由一层极薄的壳（Harness）来串联。模型只需要懂得阅读场景并按方法执行即可，系统因此能够不断自我进化、永不衰退。

## 框架本身就是产品
2026年3月，Anthropic 意外泄露了 Claude Code 的全部源码。51.2 万行代码印证了 Garry 在 YC 所教的观点：秘密不在于大模型本身，而在于包裹在它外面的那层东西。
这层包裹包括：实时仓库上下文、提示词缓存、专用工具、减少上下文臃肿、结构化会话记忆、并行子代理等。它们本身不能让模型变得“更聪明”，但能让模型在正确的时间获得正确的上下文信息，而不是用一堆噪音把它淹没。这层包裹被称之为“框架”（Harness）。所有 AI 构建者都该问一个问题：哪些该放进框架，哪些不该放？答案就是“薄框架，厚技能”。

## 定义清晰的五大概念
瓶颈从来不是大模型的智力，它们懂得推理、综合和写代码，它们失败往往是因为不理解你具体的业务模式或数据类型。修复这个问题需要以下五个概念：

### 1. 技能文件 (Skill files)
一个技能文件是一份可复用的 Markdown 文档，教会模型“怎么做”（How）。它就像是一个方法调用，接受各种参数，不同的参数通过同一套方法会产生完全不同的能力。
这就不是简单的“提示词工程”，而是纯粹的“软件设计”，使用 Markdown 作为编程语言，人类判断作为运行环境。由于它直接以模型思考的语言编写了过程、判断和上下文，因而比僵硬的源代码能更完美地封装能力。

### 2. 框架 (The harness)
框架负责运行大模型，它只做四件事：循环运行模型、读写文件、管理上下文、强制执行安全校验。这就是所谓的“轻”。
最糟糕的模式是“重框架，轻技能”，比如定义了 40+ 个繁琐的工具定义吃掉了一半的上下文窗口，每次调用耗时极高。软件不需要那么娇贵，完全可以量身打造速度极快的专用小型 CLI 工具，而不是每次都靠重型的大框架去包揽一切。

### 3. 解析器 (Resolvers)
解析器是上下文的“路由表”。它告诉系统当出现什么类型的任务时，应该优先加载哪份文档。例如，Claude Code 就内置了解析器机制，只要匹配用户的意图（技能描述字段），自动为你加载对应技能的上下文，防止了几万行的指导文件污染了上下文的窗口，造成模型注意力下降。

### 4. 潜在空间 vs. 确定性 (Latent vs. deterministic)
系统里的每一个步骤只属于这两种类别之一，搞混它们是 Agent 设计中最常见的错误。
- **潜在空间（Latent space）**：承载智能（Intelligence）的地方。阅读、解释、决定、判断、模式识别。
- **确定性（Deterministic）**：承载信任的地方。同一输入必然得到同一输出。如 SQL 查询、编译代码、算术。
最优秀的系统在这个问题上是非常无情的，绝不会把组合优化（比如给800人排座位）这样的硬逻辑错误地交给潜层空间的“猜测”去做。

### 5. 日志化/结构化沉淀 (Diarization)
这一步让 AI 在知识工作上真正有用。大模型去阅读几十乃至上百份分散的文件，并在“记住冲突”的情况下，综合提炼出一份单页的、充满判断力的结构化画像。这种操作是传统数据库 lookup 或 RAG 流水线都做不到的，只有模型才能综合阅读后生成一份精炼的简报。

## 系统的演进与三层架构
这五个概念组成了一个简单的三层架构：
- **顶层是重技能 (Fat Skills)**：编写有判断力、流程和领域知识的 Markdown 文件。这里蕴含了 90% 的价值。
- **中间层是轻框架 (Thin Harness)**：极简的代码，接收 JSON 并输出文本，默认只读。
- **底层是应用 (Your application)**：查询数据库、读取文档、搜索等确定性的基础工具。
其原则是有方向性的：把智能向上推给技能，把执行向下推给确定性工具，将框架留得很薄。如此一来，当新的大模型发布时，所有技能会随之自动升级，而底层的确定性应用依然可靠。

## 不断学习的系统
Garry 用一个为 6000 个 YC 创业者精准匹配的真实例子展现了这个系统的威力：
1. **数据丰富 (Enrichment)**：通过技能 `/enrich-founder`，模型会阅读创始人的陈述与 GitHub 提交记录之间的“落差”，做出哪怕用关键词过滤也无法达成的深度判断，并将这类操作全部放在“潜在空间”中执行。
2. **场景匹配 (Matching)**：使用技能调用的方式，在不同场景（如按标签匹配、随机发现匹配、实时配对）下发挥了极高的判断力。传统算法无法识别两人虽然都在做“AI 基础架构”，但一人是做计费，另一人是做调度，不会产生竞品冲突这种隐性关联。
3. **闭环学习 (The learning loop)**：活动结束后，用 `/improve` 技能自动读取用户中评反馈、找出系统的偏差，并**自动提议新规则写回到之前的匹配技能文件中**。这意味着下一次执行时，系统不需要任何人改动代码就变得更精准了，技能文件完成了自我更新与重写！
获取、阅读、沉淀、计数、综合；然后：调查、调查、沉淀、重写技能。这是 2026 年最有价值的工作闭环。

## 技能是永久的升级
如果你要求 AI 做一件“会重复发生”的事，绝不允许让它变成一次性的活。要求 AI 试做几次，确认无误后将其“固化”为一个技能文件，甚至挂载到 Cron 定时任务里。“如果我必须向你要求同一件事两次，你就失败了。”
每一次固化一个技能，都是对整个系统进行了一次“永久性升级”。它永远不会退化、不会遗忘，会在半夜默默运行；并且新一代模型诞生时，系统立刻获得成倍的威力——潜在空间的判断力在变强，而确定性的步骤永远稳定不崩。
这就是如何让生产力达到那所谓的 100 倍。构建一次，永远运行。
