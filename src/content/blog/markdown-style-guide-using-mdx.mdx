# Markdown Style Guide

This guide provides an overview of the supported Markdown and MDX syntax for this blog.

## Common Markdown Elements

### Headers
Use `#` to `######` for different levels of headers.

### Emphasis
- *Italic* or _Italic_
- **Bold** or __Bold__
- ***Bold and Italic***

### Lists
1. First item
2. Second item
   - Sub-item
   - Sub-item

### Links and Images
[OpenClaw](https://openclaw.ai)
![Placeholder Image](/blog-placeholder-1.jpg)

### Blockquotes
> "Efficiency is doing things right; effectiveness is doing the right things." - Peter Drucker

### Code Blocks
```javascript
function greet(name) {
  console.log(`Hello, ${name}!`);
}
```

## MDX Special Features

MDX allows you to use React components directly within your Markdown.

### Importing Components
You can import components at the top of the file (after the frontmatter).

```javascript
import CustomComponent from '../../components/CustomComponent.astro';
```

### Using Components
<div style={{ padding: '20px', backgroundColor: 'tomato', color: 'white' }}>
  This is a div styled with JSX inside Markdown!
</div>
